Reverse-proxy map (Caddy)

Expose only what you intend. Keep private by omitting DNS and proxy blocks.

Examples (add to /etc/caddy/Caddyfile) 
# Static site (already live)
loganlancer.com, www.loganlancer.com {
  root * /var/www/main
  encode zstd gzip
  file_server
}

# Optional app subdomains (enable one at a time; set DNS first)
vpn.loganlancer.com {
  reverse_proxy 192.168.1.X:1194    # if you terminate OpenVPN via TCP+Web, otherwise port-forward 1194/UDP
}

dash.loganlancer.com {
  reverse_proxy 192.168.1.X:7575    # Homarr
  header { Strict-Transport-Security "max-age=31536000; includeSubDomains" }
  basicauth /* {                          # optional extra guard
    {$DASH_USER} {$DASH_PASS_HASH}
  }
}

media.loganlancer.com {
  reverse_proxy 192.168.1.X:8096    # Jellyfin HTTP
}

ai.loganlancer.com {
  reverse_proxy 192.168.1.X:11434   # Ollama API
}

voice.loganlancer.com {
  reverse_proxy 192.168.1.X:PORT    # Piper/Whisper gateway when ready
}

For services that should stay private, do not create the DNS record and do not add a proxy block.

# Allow 80/443 to 301 (webgw) if PVE FW is enabled

LXC UFW

# on each container
ufw allow 80,443/tcp          # webgw
ufw allow 7575/tcp            # homarr (if proxied)
ufw allow 8096,8920/tcp       # jellyfin (if proxied)
ufw allow 11434/tcp           # ollama (if proxied)
ufw allow 1194/udp            # openvpn (if direct)
