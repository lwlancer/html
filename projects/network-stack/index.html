Node and Service Roles

LETOII — Primary host for user-facing dashboards and entertainment services.

  • VM 103 “bellonda” — Runs HomeAssistant, a FOSS smart home administrator that I have ready to replace my Alexas once "siona" is working.
  • VM 501 “teg” — Dedicated Minecraft server. Serves a persistent survival world for my friends and I.

ALIA — AI and voice compute node for local models and inference.

  • LXC 999 “murbella” — Hosts the Ollama LLM stack. Provides text generation APIs used by internal tools and hopefully soon my smart home system.
  • LXC 909 “siona” — Reserved for Piper (TTS) and Whisper (STT). Will handle local voice synthesis and transcription when linked with Ollama and the dashboard.

GHANIMA — Networking and public gateway node.

  • LXC 102 “lucilla” — Runs WireGuard, providing secure access to the entire LAN and Proxmox network over a lightweight VPN tunnel. May switch this to a cloudflare solution soon, but WireGuard has just never let me down.
  • LXC 301 “sheeana” — Hosts Apache for static site delivery and cloudflared for outbound tunneling to Cloudflare. Where this website lives!

Datacenter orchestration — The Arrakis cluster handles HA quorum, local backups, and monitoring. Each LXC or VM can communicate securely across the internal VLAN without touching the router.

Cloudflare & Public Access

DNS (Cloudflare)

loganlancer.com     CNAME   <tunnel-uuid>.cfargotunnel.com   Proxied
www.loganlancer.com  CNAME   <tunnel-uuid>.cfargotunnel.com   Proxied
# Nameservers: Cloudflare
# SSL/TLS: Full (edge terminates TLS)
# Always Use HTTPS: On

Tunnel config (origin HTTP)

tunnel: webserver
credentials-file: /root/.cloudflared/webgw.json
ingress:
  - hostname: loganlancer.com
    service: http://localhost:80
  - hostname: www.loganlancer.com
    service: http://localhost:80
  - service: http_status:404

Apache vhost (origin)

<VirtualHost *:80>
  ServerName loganlancer.com
  ServerAlias www.loganlancer.com
  DocumentRoot /var/www/html
  # No HTTPS redirects; Cloudflare handles TLS and canonicals.
</VirtualHost>
